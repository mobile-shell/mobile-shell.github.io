Mosh

(mobile shell)

Remote terminal application that allows roaming, supports intermittent connectivity, and provides intelligent local echo and line editing of user keystrokes.

Mosh is like SSH, but more robust and responsive — especially over wireless or cross-country links and ones with packet loss.

Mosh is free software, available for GNU/Linux and Mac OS X.

Getting Mosh »

Change IP address. Keep your connection.

Mosh won't die when your laptop roams from one Wi-Fi network to another, or from Wi-Fi to a cellular data connection to wired Ethernet. Go on a trip and visit five hotels and three airports: your connection will automatically follow you.

Lose the Internet without losing your session.

With Mosh, you can put your laptop to sleep and wake it up later, while keeping your connection intact. If your Internet connection drops, Mosh will warn you that it hasn't heard from the server in a while — but the connection will resume as soon as network service does.

Get rid of network lag.

Mosh intelligently echoes keystrokes and some line editing at the client side, so your typing usually shows up on the screen instantly. No more waiting for your packets to get to the server and back before you see what you typed or where the cursor has landed. Mosh handles lossy links gracefully, unlike TCP-based protocols.

No privileged code. No daemons.

You don't need to be the superuser to install or run Mosh. The client and server are executables run by an ordinary user and last only for the life of the connection.

Keep your remote-access method.

Mosh doesn't listen on network ports or authenticate users. Users keep the existing remote-access mechanism, probably SSH, and present the same credentials (e.g. password, public key). The mosh client takes care of logging in to the server via SSH and then runs the mosh-server program remotely.

Runs inside your terminal emulator, but with better Unicode.

Mosh is a command-line program, just like ssh. You can use it inside xterm, gnome-terminal, rxvt-unicode, Terminal.app, or emacs ansi-term. But mosh was built from scratch to support UTF-8 as its only encoding, and fixes Unicode problems in other terminals and in ssh.

Control-C works great.

Mosh works at the terminal emulation layer and uses its own flow control to set the frame rate based on network conditions. It doesn't fill up network buffers, so Control-C always works instantly (within an RTT) to halt a runaway process.
