Mosh

(mobile shell)

Remote terminal application that allows roaming, supports intermittent connectivity, and provides intelligent local echo and line editing of user keystrokes.

Mosh is like SSH, but more robust and responsive — especially over wireless or cross-country links and ones with packet loss.

Mosh is free software, available for GNU/Linux and Mac OS X.

Getting Mosh »

Change IP address. Stay connected.

Mosh automatically roams as you move between Internet connections. Use Wi-Fi on the train, Ethernet in a hotel, and LTE on the beach. You'll stay logged in.

Lose the Internet without losing your session.

With Mosh, you can put your laptop to sleep and wake it up later, keeping your connection intact. If your Internet connection drops, Mosh will warn you — but the connection will resume as soon as network service does.

Get rid of network lag.

Mosh's intelligent local echo gives lag-free typing, by predicting the effects of keystrokes before the server acknowledges them. It works at the command line and in full-screen programs like emacs and vim. On a bad connection, outstanding predictions are underlined so you aren't misled.

No privileged code. No daemons.

You don't need to be the superuser to install or run Mosh. The client and server are executables run by an ordinary user and last only for the life of the connection.

Keep your remote-access method.

Mosh doesn't listen on network ports or authenticate users. Users start their sessions with SSH and present the same credentials (e.g. password, public key, keyfob). The mosh client logs in to the server via SSH and runs the mosh-server remotely.

Runs inside your terminal, but better.

Mosh is a command-line program, just like ssh. You can use it inside xterm, gnome-terminal, rxvt-unicode, Terminal.app, emacs ansi-term, screen, or tmux. But mosh was designed from scratch to support UTF-8, and compensates for Unicode bugs in other terminals and in ssh.

Control-C works great.

Unlike SSH, mosh's UDP-based protocol handles packet loss gracefully, and sets the frame rate based on network conditions. Mosh doesn't fill up network buffers, so Control-C always works within an RTT to halt a runaway process.

Getting Mosh

Debian GNU/Linux unstable

# apt-get install mosh

Ubuntu 10.04 LTS or later

$ sudo add-apt-repository ppa:keithw/mosh
$ sudo apt-get update
$ sudo apt-get install mosh

MacPorts OS X 10.6 or later

$ sudo port install mosh

Homebrew OS X 10.6 or later

$ brew install mobile-shell

Static server binary

If you cannot install a Mosh package on the server, you can put a statically-linked mosh-server binary in your home directory.

You will need to launch mosh as

$ mosh --server=./mosh-server
